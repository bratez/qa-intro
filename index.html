Что такое DOM:

Основным инструментом работы и динамических изменений на странице является DOM (Document Object Model) – объектная модель, используемая для XML/HTML-документов. Согласно DOM-модели, документ является иерархией, деревом. Каждый HTML-тег образует узел дерева с типом «элемент». Вложенные в него теги становятся дочерними узлами. Для представления текста создаются узлы с типом «текст». DOM – это представление документа в виде дерева объектов



Что такое теги:

p, div, span, a, table, ul, img, audio, video, svg, custom-tag

*стили по умолчанию



Как тегам задаются стили:

  • 1. С помощью подключения файла стилей в теге head страницы:

    <link rel="stylesheet" href="styles.css">

  • 2. С помощью добавления стилей в специальный тег:

    <style>
    p {
        font-style: italic;
    }

    span {
        display: block;
    }
</style>

  • 3. С помощью добавления стилей "инлайново" в атрибут "style" каждому элементу:

    <p style="background-color: black; color: #fff;">Текст для примера</p>


Приоритетность стилей:

<p class="example-text" id="example_text_id" style="background-color: black; color: #fff;">Текст для примера</p>

<style>
    p.example-text {
        text-transform: uppercase;
    }

    .example-text {
        text-transform: lowercase;
    }
</style>

XPath

Цель CSS 3 XPath
Все элементы * //*
Все параграфы p //p
Все дочерние элементы p > * //p/*
Элемент по идентификатору #foo //*[@id=’foo’]
Элемент(ы) по классу .foo //*[contains(@class,’foo’)] 1
Элементы по имени атрибута *[title] //*[@title]
Первые дочерние элементы в параграфах p > *:first-child //p/*[0]
Все ссылки в параграфах Not possible //p[a]
Следующий элемент p + * //p/following-sibling::*[0]


XPath vs JavaScript vs Sizzle vs jQuery

  • XPath - .//*[contains(@class,'head')]
  • JS - document.querySelectorAll('div')
  • Sizzle - Sizzle('.ololo')
  • jQuery - $('.trololo')


Установка Selenium2Library и запуск тестового проекта:

На компьютере должен быть установлен Phyton с "pip"

Для работы с браузером FireFox в OS Windows может понадобится установка драйвера geckodriver, для этого:

  • Качаем последюю версию дравера отсюда
  • Копируем файл из архива, к примеру, C:\Users\USERNAME\geckodriver\
  • Добавляем этот путь в переменную PATH системы как показано тут

В консоли в нужной директории (C:\Users\USERNAME\) запускаем команды:

pip install robotframework
pip install robotframework-selenium2library

Качаем тестовый проект (или клонируем с помощью git'a) здесь и распаковываем в определенную папку

В консоли переходим в папку webdemo тестового проекта и запускаем тестовый проект:

python demoapp/server.py

и запускаем тестирование:

python -m robot.run login_tests/
